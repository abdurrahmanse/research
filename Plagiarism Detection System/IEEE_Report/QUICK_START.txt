OPTION 2: Local LaTeX Compilation
──────────────────────────────────
Prerequisites (if not installed):
  - macOS:  brew install mactex
  - Ubuntu: apt-get install texlive-full
  - CentOS: yum install texlive

Compilation Commands:

  cd IEEE_Report
  
  # Method 1: Standard three-pass compilation
  pdflatex main.tex && bibtex main && pdflatex main.tex && pdflatex main.tex
  
  # Method 2: Using latexmk (automatic, recommended)
  latexmk -pdf main.tex
  
  # Clean auxiliary files
  rm -f main.{aux,bbl,blg,fls,fdb_latexmk,log,out}

Output: main.pdf (12-15 pages for 11-section format)

STATISTICS & INSIGHTS
================================================================================

Study Dataset:
  - 100+ academic text pairs
  - 50% plagiarized, 50% original (balanced)
  - Domain: AI/ML research papers
  - Diversity: 4 plagiarism type variants

Training Configuration Per Model:
  - TF-IDF: Immediate deployment (no training)
  - BiLSTM: 5-10 minutes training time
  - BERT: 15-30 minutes fine-tuning time

Type-Specific Detection Rates:
  - Direct Copying: 95%+ (all models excellent)
  - Paraphrasing: BERT 88.6% >> BiLSTM 78.9% >> TF-IDF 62.1%
  - Semantic Variation: BERT 91.2% >> BiLSTM 84.3% >> TF-IDF 71.8%
  - Gap demonstrates semantic understanding importance (26.5 percentage point spread)

Generalization Analysis:
  - In-domain (AI/ML papers): Baseline performance
  - Cross-domain (Biology/Chemistry): -2% to -4% accuracy drop
  - Technical documentation: -4% to -8% accuracy drop
  - BERT shows best domain transfer, TF-IDF most stable
